# Platform Context — Семейная платформа изучения английского

*Главный стратегический документ платформы. Читается рядом с системными промптами наставников. Обновляется при изменении стратегических решений, не при операционных правках.*

*Версия: 1.0 · Дата: 2026-05-27*

---

## 1. Что это

Семейная веб-платформа изучения английского языка на 4 пользователей (отец, мама, Матвей, Никита). Разрабатывается Матвеем (15 лет) как сквозной учебный проект в треке наставника. Семья получает работающий инструмент как побочный результат.

Рамка заказчика: **«учебный проект Матвея, побочным результатом которого семья получает инструмент изучения английского»**.

Платформа не заменяет существующее обучение в EnglishDom и Green Country, а усиливает его — закрепляет материал между уроками, даёт независимую диагностику уровня, переводит уже работающую семейную практику ежедневных отчётов в цифровой формат.

---

## 2. Четыре приоритетные ценности

В порядке убывания приоритета. При конфликте — выигрывает более ранняя.

**1. Прозрачность процесса.** Независимая диагностика реального уровня каждого члена семьи. Закрывает давнюю боль заказчика: «6 лет занимался в Green Country, не понимал реального уровня сына». Платформа показывает, что именно человек знает и не знает, без интересов школы или сервиса.

**2. Учебный проект Матвея.** Рост Матвея как разработчика через работу над реальным продуктом с реальными пользователями. Платформа — основной учебный контекст наставника. Все остальные ценности подчинены этой.

**3. Усиление существующего обучения в EnglishDom.** Закрепление и практика между уроками. Не конкурент EnglishDom, а его продолжение.

**4. Экономия в будущем.** Горизонт от 12 месяцев. Постепенное снижение зависимости от внешних подписок по мере роста собственных модулей. Сегодня — не цель, а отложенный бонус.

---

## 3. Для кого

**Активные пользователи (с первого дня):**
- Отец — EnglishDom, 2 индивидуальных урока в неделю
- Мама — EnglishDom, 2 индивидуальных урока в неделю
- Матвей (15) — летом отдыхает от занятий, осенью индивидуальные в EnglishDom
- Никита — самостоятельные занятия по бесплатным курсам EnglishDom, ежедневный отчёт в семейный чат

**Существующий контекст семейной практики:** общий Telegram-чат, дети присылают вечерний отчёт о проделанной работе, мама видит, соревновательный момент работает. Семейный слой платформы цифровизирует уже работающую практику, а не строит её с нуля.

---

## 4. Архитектурные принципы

Эти принципы — стратегические рамки. Технические решения внутри них принимает Матвей с помощью своего рабочего AI.

**Единое ядро, растущее постепенно.** Не event-sourcing с проектированием на все модули наперёд. Минимальное ядро с первого модуля, расширяется с каждым новым. Метафора: ядро как единый мозговой центр, который видит каждого пользователя и семью целиком.

**Грамматика как сквозной слой.** Не отдельный модуль. Таблица `grammar_topics` в ядре с первого дня. Все события (карточки, Speaking-ошибки, чтение) тегируются грамматическими темами. Дашборд грамматики появляется естественно к 3–4 месяцу, когда накоплены данные.

**Трёхуровневая видимость данных (с первого дня):**

- **Личный дашборд** — пользователь видит про себя, для самоулучшения
- **Семейный фид** — все видят всех, только мотивирующая информация: новые слова, активность, прогресс. Цифровизация семейного чата
- **Родительский дашборд** — полная картина по детям, только отцу и маме

Защита приватности Матвея: в семейном фиде не больше, чем в текущем Telegram-чате семьи. Детальная диагностика и ошибки — только родительский уровень.

**Один бот — два наставника.** Общая инфраструктура (код, БД, хостинг, API). Раздельные системные промпты и раздельные учебные карты для Матвея и Никиты. Не условные ветви в одном промпте.

---

## 5. Технологический стек

Решения по стеку приняты исходя из текущих навыков Матвея и принципа минимализма.

- **Бэкенд:** FastAPI
- **Фронтенд:** серверный рендеринг через Jinja2. Не SPA, не React. Минималистичный UX
- **БД:** SQLite на старте, PostgreSQL по необходимости
- **Интерфейс:** веб-приложение с авторизацией и личными кабинетами
- **Алгоритм повторения:** готовый Python-пакет `fsrs`
- **AI:** Claude API с prompt caching
- **Голос (для Speaking):** Whisper API + ElevenLabs/Cartesia или OpenAI TTS

Стек — стартовая база, не догма. По мере роста Матвея и платформы решения пересматриваются.

## 6. Роли

**Отец** — продакт-заказчик. Прямой контакт с Матвеем как с разработчиком. Принимает приоритеты, тестирует с семьёй, защищает от feature creep.

**Наставник Матвея (AI).** Педагогический партнёр в проекте платформы. **Не пишет код**, не принимает технических решений. Помогает Матвею расти как разработчику через работу над платформой. Ведёт еженедельный отчёт по воскресеньям. Правка промпта — раз в месяц планово; внепланово — только по инициативе наставника при серьёзной причине, заявленной в отчёте.

**Матвей** — разработчик. Технические решения принимает сам с помощью своего рабочего AI (Claude Code, ChatGPT). Наставник в эти решения не вмешивается.

**Наставник Никиты (AI).** Отдельный персонаж в общем боте. Появится через 2–3 недели, после профилирования Никиты. До этого момента темп Матвея с платформой важнее.

**Claude (стратегический партнёр отца).** Помогает отцу как продакту: формулировать приоритеты, держать целостность видения, готовить артефакты проекта. Не общается с Матвеем напрямую.

## 7. Последовательность модулей

Порядок определён ценностями (раздел 2): что раньше закрывает приоритетную ценность — то раньше реализуется.

| Этап | Модуль | Какую ценность закрывает |
|---|---|---|
| 0+1 | Ядро + Карточки FSRS с удобным вводом из уроков EnglishDom | Усиление EnglishDom: немедленное закрепление слов из живых уроков |
| 2 | Speaking | Закрывает разговорную практику; поддерживает Матвея летом без EnglishDom |
| 3 | Writing | Та же архитектура, что Speaking |
| 4 | Дашборд CEFR | Первая видимая реализация прозрачности |
| 5 | Чтение | Расширяет вход контента |
| 6 | Семейный слой полный | Расширение базовой видимости |
| 7 | Аудирование | Дополняет |
| 8 | Видео через Language Reactor | Главный канал погружения |
| 9 | Произношение | Когда подоспеет технология |

Последовательность — рабочая гипотеза. Корректируется по мере работы, но не под давлением сиюминутных идей.

---

## 8. Что платформа НЕ делает

Эти границы зафиксированы, чтобы защититься от feature creep.

- **Не заменяет EnglishDom и Green Country.** Усиливает, не конкурирует.
- **Не создаёт мотивацию с нуля.** Цифровизирует уже работающую семейную практику.
- **Не пишет код за Матвея.** Наставник — педагог, не co-pilot.
- **Не масштабируется за пределы семьи.** Не SaaS, не стартап, не продукт для рынка. 4 пользователя — потолок проектирования.
- **Не оптимизируется под скорость поставки.** Это учебный проект; темп задаёт обучение Матвея, не дедлайны.

## 9. Цикл отчётности и пересмотра

**Еженедельный отчёт наставника Матвея** — каждое воскресенье, полная структура из 10 пунктов (определена в системном промпте).

**Плановая правка промпта наставника** — раз в месяц. Защита от over-fitting на недельные колебания.

**Внеплановая правка** — только по инициативе наставника, при серьёзной причине, заявленной в еженедельном отчёте.

**Пересмотр этого документа** — при изменении стратегических решений (ценности, архитектурные принципы, роли, последовательность модулей). Не при операционных правках. Старые версии — в `/archive`.
