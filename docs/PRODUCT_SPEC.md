# Family English — продуктовая спецификация

**Версия:** 1.0 (рабочая, собрана из всех исходников) · **Дата:** 2026-06-21
**Назначение:** единый источник правды по продукту. Собран из `platform_context.md`,
`family_ai_tutor_vision.docx` и `session_summary_2026-05-27.md` (лежат в `docs/source/`).

> **Смена концепции (2026-06-21).** Раньше платформа была «учебным проектом Матвея,
> а инструмент для семьи — побочный результат». **Теперь наоборот:** Family English —
> это **продукт для семьи, который запускаем сейчас**. Развитие Матвея как разработчика —
> **отдельный следующий проект**. Всё «наставническое» вынесено из этой спеки (см. `SOURCES.md`).

---

## 1. Одна идея

Семейная веб-платформа изучения английского для 4 человек. Главная ценность — **не AI
сам по себе, а единая память обучения**: слово, на котором запнулся в чтении или видео,
автоматически попадает в карточки на повторение, а через неделю всплывает в разговоре с AI.
Разрозненные Anki + YouTube + ChatGPT так не умеют — они не общаются между собой.

## 2. Для кого

| Пользователь | Уровень / контекст |
|---|---|
| Папа | EnglishDom, 2 индивид. урока/нед |
| Мама | EnglishDom, 2 индивид. урока/нед |
| Матвей (15) | Reading сильный (~C2), летом без занятий |
| Никита (11) | начинающий, ежедневный отчёт в семейный чат |

Не SaaS, не для рынка. **4 пользователя — потолок проектирования.** Auth простой,
платежей нет, масштабирования на тысячи нет.

## 3. Ценности (порядок после смены концепции)

1. **Прозрачность уровня** — независимая диагностика, что человек реально знает и не знает.
2. **Усиление EnglishDom** — закрепление слов из живых уроков между занятиями.
3. **Семья реально пользуется каждый день** — продукт должен возвращать к себе (раньше это
   было побочным; теперь это критерий успеха).
4. **Экономия в будущем** — горизонт 12+ мес, меньше внешних подписок. Отложенный бонус.

## 4. Ядро данных («единый мозг»)

Принцип: **минимальное ядро, растущее с каждым модулем.** Не проектируем все таблицы наперёд.
Любой модуль умеет писать «событие изучения», другие модули его читают.

**Уже построено (v1, модуль карточек):**
- `users` — профиль, CEFR, дневная цель, пароль (хэш)
- `words` — общий каталог слов (front/back/level), ~214 слов A1–C2
- `cards` — карточки ученика + состояние FSRS
- `learning_events` — некорректируемый журнал повторений (append-only)
- `grammar_topics` + тег грамматики на словах/карточках/событиях — **сквозной слой**, не
  модуль. Дашборд грамматики «вырастает сам» из накопленных тегов (к 3–4 месяцу)
- **Ввод слов из уроков EnglishDom** — страница `/my-words`: ученик вставляет список
  `слово | перевод`, всё сразу становится карточками (с защитой от дублей)

**Появятся вместе со своими модулями (не раньше):**
- `mistakes` (original/correction/category/source_module) — со Speaking/Writing
- `sessions` (общая модель сеанса любого модуля) — со Speaking
- `content_items` (тексты/аудио/видео с разметкой по уровню) — с Чтением
- `family_shared` (общие колоды, темы, достижения) — с семейным слоем

## 5. Модули и порядок запуска

Порядок сведён из двух источников (спека отдаёт приоритет карточкам ради EnglishDom; виженис
ставит Speaking первым ради «вау»). С учётом смены концепции «запускаем уже» и того, что
**ядро+карточки уже готовы**, порядок такой:

| # | Модуль | Статус | Зачем |
|---|---|---|---|
| 0+1 | **Ядро + Карточки (FSRS)** | ✅ ГОТОВО | Запускаем семье сейчас. Закрепление слов из EnglishDom |
| 2 | **Speaking** — голосовой собеседник AI | следующий | Главный «вау» и драйвер возвращаемости. Push-to-talk, разбор ошибок |
| 3 | **Чтение** со встроенным переводом | далее | Тап по слову → перевод → «в карточки» |
| 4 | **Письмо** | далее | Та же архитектура, что Speaking, но текст |
| 5 | **Дашборд CEFR + семейный фид** | далее | Видимая «прозрачность» + цифровизация семейного чата |
| 6 | Аудирование | позже | Короткие аудио под уровень |
| 7 | **Видео** (mp4+SRT → потом YouTube) | позже | Главный канал погружения. Самый сложный — в конце |

Правило: **каждый модуль выкатывается на боевой отдельно, когда им реально начинают пользоваться.**
Не «весь проект целиком».

## 6. Три уровня видимости данных (закладываем с первого дня)

- **Личный дашборд** — человек видит про себя.
- **Семейный фид** — все видят всех, только мотивирующее (новые слова, активность). Цифровизация
  уже работающего семейного Telegram-чата.
- **Родительский дашборд** — полная картина по детям, только папе и маме. Детальная диагностика
  и ошибки — только здесь (приватность детей).

## 7. Стек

- Бэкенд: **FastAPI**. Фронт: **серверный рендеринг (Jinja2)**, не SPA, минималистичный UX.
- БД: **SQLite сейчас → PostgreSQL по необходимости** (переключается переменной `DATABASE_URL`).
- Повторение: готовый пакет **`fsrs`** (не пишем с нуля).
- AI: **Claude API + prompt caching** (Sonnet для педагогики, Haiku для простого — перевод слова).
- Голос (для Speaking): **Whisper API** (STT) + **ElevenLabs/Cartesia или OpenAI TTS**; Web Speech
  API в браузере — бесплатный старт.
- Хостинг: **Docker-контейнер на нашем Hetzner-VPS** рядом с ERP, за Nginx, на поддомене.
- Безопасность: пароли хэшируются, секреты только в `.env`, сессии в подписанной cookie.

## 8. Границы и стоимость

**Не делаем:** не заменяем EnglishDom/Green Country (усиливаем); не строим мотивацию с нуля
(цифровизируем семейную практику); не SaaS; не гонимся за скоростью в ущерб тому, чтобы семья
реально пользовалась.

**Стоимость эксплуатации** (когда подключим AI-модули): ~**$20–40/мес** на API при агрессивном
кэшировании и Haiku для простых задач. Карточки (v1) — **бесплатно**. Локальная модель (Ollama) —
не нужна, требует дорогого железа; опциональный апгрейд через год.
